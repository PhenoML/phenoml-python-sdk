## 9.5.0 - 2026-04-02
* The SDK now retries requests on network-level connection errors (`ConnectError`, `RemoteProtocolError`) using exponential backoff with jitter, in addition to the existing server-error retry behavior. A new `ParsingError` exception is raised when a server response cannot be deserialized, providing clearer diagnostics. The minimum supported Python version has been raised to **3.10**; users on Python 3.8 or 3.9 must upgrade their runtime to use this version of the SDK.
* The SDK now raises a new `ParsingError` exception (available from `phenoml.core`) when a server response cannot be validated against the expected schema. This replaces uncaught Pydantic `ValidationError` exceptions and provides structured access to the HTTP status code, response headers, raw body, and the underlying cause, making it easier to diagnose unexpected API responses.
* The SDK now raises a `ParsingError` (instead of an unhandled `pydantic.ValidationError`) when a response cannot be deserialized. This provides a consistent error type with HTTP status code, headers, and body context across all client methods in the `lang2fhir`, `summary`, `tools`, and `mcp_server` modules.
